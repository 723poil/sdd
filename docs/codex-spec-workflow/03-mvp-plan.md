# MVP Plan

## MVP 범위

첫 구현은 "코드를 자동 수정하는 앱"이 아니라 "로컬 프로젝트를 읽고 명세를 정리하는 앱"에 집중한다.

MVP의 끝점은 "Codex가 실제 코드를 수정하는 것"이 아니라 "명세와 프로젝트 문맥을 정리해 다음 작업에 넘길 준비를 마치는 것"이다.

### 포함

- 로컬 UI 실행
- 앱 전역 CLI 에이전트 연결 설정
- 로컬 프로젝트 폴더 선택
- 프로젝트 분석
- 프로젝트 개요 생성
- 명세 초안 자동 생성
- 명세서 목록/조회/저장
- 명세서별 채팅
- 채팅 기반 수정안 생성
- diff 표시
- 명세 반영
- 로컬 저장 및 재진입

### 제외

- 다중 사용자 협업
- 클라우드 동기화
- 복잡한 권한 모델
- 고급 영향 분석
- Codex 실제 코드 수정 자동화
- 테스트 자동 실행
- Git commit 연동

## MVP 화면 흐름

### 0. 앱 설정 화면

- CLI 에이전트 연결 설정
- 현재는 `Codex CLI` 우선 노출
- PATH 자동 감지 또는 직접 실행 경로 지정
- 인증 방식 기록
- 연결 확인

주의:

- 이 화면은 실제 분석/채팅 실행과 분리한다
- 프로젝트 내부 `.sdd/`가 아니라 앱 전역 설정 파일을 사용한다

### 1. 시작 화면

- 최근 프로젝트 목록
- `프로젝트 폴더 선택`
- 빈 상태 안내

참고:

- 최근 프로젝트 목록은 프로젝트 내부 `.sdd/`가 아니라 앱 전역 설정 파일로 관리한다.

### 2. 프로젝트 선택 / 가져오기

- 로컬 폴더 선택
- 경로 검증
- 기본 정보 표시
- `분석 시작`

### 3. 분석 진행 화면

- 파일 스캔
- 스택 감지
- 엔트리포인트 탐지
- 주요 파일 추출
- 요약 생성

### 4. 프로젝트 개요 화면

- 감지된 스택
- 주요 디렉터리
- 핵심 모듈
- 엔트리포인트 후보
- unknowns / confidence
- `명세 초안 생성`
- `다시 분석`

### 5. 명세 작업 화면

- 명세 목록
- 선택한 명세 본문
- 명세 채팅
- `새 명세서`
- `초안 다시 생성`
- `변경 diff 보기`
- `명세 반영`

### 6. 명세 변경 검토 화면

- 변경 전/후 비교
- 변경 요약
- `반영`
- `보류`
- `다시 요청`

### 7. 작업 준비 화면

- 현재 기준 명세
- 관련 파일 후보
- 프로젝트 문맥 요약

초기 MVP는 여기까지 만들면 충분하다.

## 1차 기능 목록

### 반드시 포함

- 프로그램 실행 후 로컬 UI 표시
- 로컬 프로젝트 폴더 선택
- 파일 스캔
- 언어/프레임워크 감지
- 주요 파일 및 디렉터리 요약
- 프로젝트 개요 생성
- 명세 초안 자동 생성
- 명세서 저장
- 명세서 채팅
- 수정안 생성
- diff 보기
- 수정안 반영
- 분석 결과와 명세서 로컬 저장

### 있으면 좋지만 미뤄도 됨

- 최근 프로젝트 목록
- 여러 프로젝트 전환
- 명세 버전 고급 비교
- 증분 재분석 최적화

## 1차 개발 순서

### Step 1. 앱 껍데기

- Electron 기본 앱
- 기본 창/UI
- 프로젝트 폴더 선택

### Step 2. 분석 파이프라인

- 파일 트리 수집
- 스택 감지
- 주요 파일 식별
- 요약 데이터 생성

### Step 3. 명세 초안 생성

- 프로젝트 개요 명세
- 구조 명세
- 기능 작업 명세

### Step 4. 명세서 채팅

- 명세별 대화 저장
- 수정안 생성
- diff 후 반영

### Step 5. 로컬 저장과 재진입

- `.sdd/` 생성
- 분석 결과 파일 저장
- 명세/채팅 재로드

### Step 6. Codex 연결 준비

- 앱 전역 CLI 연결 설정 저장
- CLI 연결 확인
- 확정 명세 + 프로젝트 컨텍스트 조립
- 다음 작업용 입력 준비

주의:

- 이 단계는 실제 코드 자동 수정이 아니라 실행 준비 단계다.
- 초기에는 설정/연결 확인까지 먼저 만들고, 분석/채팅과의 실제 연결은 뒤 단계로 둔다.
- 실제 코드 수정 자동화는 MVP 다음 단계로 둔다.

## 현재 추천 스택

- 플랫폼: `Electron`
- UI: `React + TypeScript`
- 저장: 프로젝트 내부 `.sdd/`
- 명세 저장: `markdown + json`
- 채팅 저장: `jsonl`
