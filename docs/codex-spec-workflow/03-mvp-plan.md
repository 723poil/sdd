# MVP Plan

## MVP 범위

첫 구현은 "코드를 자동 수정하는 앱"이 아니라 "로컬 프로젝트를 읽고 명세를 정리하는 앱"에 집중한다.

MVP의 끝점은 "Codex가 실제 코드를 수정하는 것"이 아니라 "명세와 프로젝트 문맥을 정리해 다음 작업에 넘길 준비를 마치는 것"이다.

### 포함

- 로컬 UI 실행
- 앱 전역 CLI 에이전트 연결 설정
- 로컬 프로젝트 폴더 선택
- 연결된 CLI 에이전트를 통한 프로젝트 분석
- 프로젝트 개요 생성
- 명세 초안 자동 생성
- 명세서 목록/조회/저장
- 채팅
- 채팅 기반 수정안 생성
- diff 표시
- 명세 반영
- 로컬 저장 및 재진입

### 제외

- 다중 사용자 협업
- 클라우드 동기화
- 복잡한 권한 모델
- 고급 영향 분석
- Codex 실제 코드 수정 자동화
- 테스트 자동 실행
- Git commit 연동

## MVP 화면 흐름

### 0. 앱 설정 화면

- CLI 에이전트 연결 설정
- 현재는 `Codex CLI` 우선 노출
- PATH 자동 감지 또는 직접 실행 경로 지정
- 인증 방식 기록
- 모델 / 추론 강도 확인 및 조정
- 연결 확인

주의:

- 이 화면은 실제 분석/채팅 실행과 분리한다
- 프로젝트 내부 `.sdd/`가 아니라 앱 전역 설정 파일을 사용한다

### 1. 시작 화면

- 최근 프로젝트 목록
- `프로젝트 폴더 선택`
- 빈 상태 안내

참고:

- 최근 프로젝트 목록은 프로젝트 내부 `.sdd/`가 아니라 앱 전역 설정 파일로 관리한다.

### 2. 프로젝트 선택 / 가져오기

- 로컬 폴더 선택
- 경로 검증
- 기본 정보 표시
- `구조화 분석 시작`

### 3. 분석 진행 화면

- 프로젝트 문맥 수집
- CLI 에이전트 입력 생성
- 구조/계층/연결성 분석
- 문서 맵 연결선과 파일별 참조 관계 수집
- 분석 산출물 저장
- 진행 상태 표시
- 분석 문서는 Markdown 기준으로 저장하고 읽는다
- 필요하면 Mermaid 로 구조도, 참조 관계, 계층도, 시퀀스, 유즈케이스를 함께 남긴다
- 분석은 activity가 이어지는 동안 계속 진행 가능해야 하며, 단순 고정 10분 wall timeout에 묶지 않는다
- 무응답이나 정체 상태만 실패로 본다
- 저장소가 크거나 역할이 분리되어 있으면 subagent 또는 delegated worker 활용을 유도하되, 사용자에게는 하나의 통합된 분석 결과만 보여준다

### 4. 프로젝트 개요 화면

- 상단 페이지 탭
  - `분석`
  - `명세`
- `분석` 페이지
  - 좌측 분석 문서 목록
  - 우측 Markdown 본문
  - 문서 맵 연결선
  - 파일별 참조 관계
  - 구조 요약 / 계층 요약 / 연결성 / 파일 인덱스
  - unknowns / confidence
  - `다시 분석`
- `명세` 페이지
  - 좌측 명세 목록
  - 우측 Markdown 본문
- 명세 단위 채팅 진입
- 오른쪽 패널은 채팅만 담당

### 5. 명세 작업 화면

- 명세 목록
- 선택한 명세 본문
- `새 명세 채팅 시작`
- 채팅
- `명세 생성`
- `초안 다시 생성`
- `변경 diff 보기`
- `명세 반영`

### 6. 명세 변경 검토 화면

- 변경 전/후 비교
- 변경 요약
- `반영`
- `보류`
- `다시 요청`

### 7. 작업 준비 화면

- 현재 기준 명세
- 관련 파일 후보
- 프로젝트 문맥 요약

초기 MVP는 여기까지 만들면 충분하다.

## 1차 개발 순서

### Step 1. 앱 껍데기

- Electron 기본 앱
- 기본 창/UI
- 프로젝트 폴더 선택

### Step 2. 분석 파이프라인

- 프로젝트 문맥 수집
- Codex exec 분석 요청 생성
- 분석 프롬프트를 XML 섹션 구조로 조립
- 구조화 결과 파싱
- `.sdd/analysis/*` 저장
- 긴 분석은 progress/activity 기반으로 추적하고, inactivity 상태일 때만 중단 처리한다
- 분석 프롬프트에는 큰 저장소에서 subagent 활용을 허용 또는 권장하는 문구를 포함한다

### Step 3. 명세 초안 생성

- 프로젝트 개요 명세
- 구조 명세
- 계층 명세
- 연결성 명세

### Step 4. 채팅

- 명세 컨텍스트별 대화 저장
- 세션은 명세 단위로 연결
- 새 명세는 별도 폼보다 `새 명세 채팅 시작`으로 초안을 연다
- 우측 채팅 입력 영역에서 현재 모델과 추론 강도를 바로 확인하고 조정
- 수정안 생성
- diff 후 반영

### Step 5. 로컬 저장과 재진입

- `.sdd/` 생성
- 분석 결과 파일 저장
- 명세/채팅 재로드
- 내부 채팅 저장 구조는 세션형 로그를 재사용할 수 있지만, 사용자에게는 그냥 채팅으로 노출

### Step 6. Codex 연결 준비

- 앱 전역 CLI 연결 설정 저장
- CLI 연결 확인
- `~/.codex/config.toml` 기본값 읽기
- 확정 명세 + 프로젝트 컨텍스트 조립
- 다음 작업용 입력 준비
- `codex exec` 기반 분석/명세 실행 경로 연결
- 장기 분석 실행 취소와 subprocess 종료 상태 반영

주의:

- 이 단계는 실제 코드 자동 수정이 아니라 실행 준비 단계다.
- 초기에는 설정/연결 확인과 분석 실행 연결을 먼저 만들고, 채팅과의 심화 연결은 뒤 단계로 둔다.
- 실제 코드 수정 자동화는 MVP 다음 단계로 둔다.
- 채팅 UI는 중앙 하단보다 우측 패널을 우선한다.

## 현재 추천 스택

- 플랫폼: `Electron`
- UI: `React + TypeScript`
- 저장: 프로젝트 내부 `.sdd/`
- 명세 저장: `markdown + json`
- 채팅 저장: `jsonl`
